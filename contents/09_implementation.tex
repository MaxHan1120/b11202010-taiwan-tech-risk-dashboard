\section{Implementation and Repository Structure}
\label{sec:implementation}

The GitHub repository contains both the report and the working prototype. The implementation is organized as follows:

\begin{table}[H]
\centering
\begin{tabular}{p{0.28\textwidth} p{0.62\textwidth}}
\toprule
\textbf{Path} & \textbf{Purpose} \\
\midrule
\texttt{backend/app} & FastAPI app, SQLite schema, ingestion modules, stock universe, and risk processing. \\
\texttt{frontend/src} & React dashboard pages and shared UI components. \\
\texttt{scripts} & Local commands for initializing the database, seeding demo data, updating data, and computing scores. \\
\texttt{tests} & Backend tests for risk score thresholds and API behavior. \\
\texttt{contents} & LaTeX report sections. \\
\texttt{.codex/skills} & Reusable semantic-layer notes for this project. \\
\bottomrule
\end{tabular}
\caption{Repository structure.}
\label{tab:repo-structure}
\end{table}

The local run path is:

\begin{enumerate}
    \item Install backend dependencies from \texttt{backend/requirements.txt}.
    \item Run \texttt{python scripts/seed\_demo\_data.py}.
    \item Run \texttt{python scripts/compute\_risk\_scores.py}.
    \item Start the API with \texttt{uvicorn backend.app.main:app --reload}.
    \item Install frontend dependencies and run \texttt{npm run dev} in \texttt{frontend}.
\end{enumerate}

The prototype was validated with backend tests and a frontend production build. The backend test suite checks that risk scores stay within expected ranges, thresholds map correctly to risk levels, the latest-risk API returns the 20-stock universe after seeding, and unknown stocks return a clear error. The frontend builds successfully with Vite.
